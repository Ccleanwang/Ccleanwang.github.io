如何使用HEXO写个人博客

常用操作

  • 启动本地服务器：hexo s
  • 新建博客文章：hexo n "文件名字"
    • 本操作会在hexo-starter\source\_posts路径下生成相应的.md文档
    • 可以直接去目标文件夹用Typora等软件对文件进行修改，在localHost服务器运行的情况下只要ctrl + s能把修改过的文章内容同步到localHost网页上
  • 生成静态网页文件:hexo g
    • hexo g 是 Hexo 命令 hexo generate 的简写，它的核心作用是 将你的 Hexo 项目中的源代码（如 Markdown 文章、模板、主题等）转换成静态网页文件，以便发布到服务器或托管平台（如 GitHub Pages、Netlify 等）
  • 远端部署：hexo d
    • 部署之前要修改好配置文件
    • 部署的目标网页要是一个有效的仓库，github/gitee都行
